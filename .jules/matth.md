## 2025-05-24 - Exact Hypergeometric Expectations for Lineage Coupling
**Learning:** Permutation testing for categorical co-occurrences without spatial dependencies (e.g. lineage coupling of clones across cell types) introduces non-deterministic variation, handles extreme tail values poorly, and is exceptionally slow. The probability of zero occurrences of a subset of cells assigned to a specific set of types within a randomly drawn clone of size $k$ perfectly maps to the hypergeometric distribution. Because calculating exact expectations and variances for single or paired presence involves evaluating probabilities of zero occurrences ($P(X = 0)$), we can use a computationally stable and fully vectorized log-gamma approximation ($P(X=0) = \exp(\ln\Gamma(N-K+1) - \ln\Gamma(N-K-k+1) - \ln\Gamma(N+1) + \ln\Gamma(N-k+1))$) and the inclusion-exclusion principle to perfectly solve for expectations and variances.
**Action:** Replace heuristic permutation testing with analytical log-gamma formulations of the hypergeometric distribution when evaluating categorical co-occurrence enrichment. It eliminates non-deterministic variation, provides exact variance estimates even in the extreme tails, and enables highly parallel vectorized calculations.

## 2025-05-25 - Exact Ordinary Least Squares for Exponential Decay
**Learning:** Fitting exponential curves ($y = a \cdot e^{bx}$) directly via non-linear least squares (e.g. `scipy.optimize.curve_fit`) relies on iterative numerical optimization. This is computationally slower, prone to local optima, and highly sensitive to initial parameter guesses and noise. Because the model is log-linear, we can apply a simple transformation ($\ln y = \ln a + bx$) to map the problem into standard Ordinary Least Squares (OLS), which has a closed-form analytical solution. This completely eliminates initialization heuristics and convergence failures, ensuring stable, exact extraction of physical attenuation parameters.
**Action:** Replace iterative numerical optimization with analytical OLS (`np.polyfit`) when fitting exponential decay models in Z-slice intensity correction. Ensure input intensities are strictly positive prior to the log transformation.

## 2025-05-25 - Architectural Rigidity and Statistical Modeling Separation
**Learning:** Hardcoding statistical fitting logic inside image processing loops creates brittle, non-extensible code and risks silent data corruption if fits fail. Furthermore, pure OLS on log-transformed data assumes multiplicative error in the original space, while Non-Linear Least Squares (NLLS) assumes additive error. Using OLS to seed NLLS guarantees convergence while maintaining the correct additive error assumption.
**Action:** Extract statistical modeling into a Strategy Pattern registry (e.g., `FIT_MODELS`). Consolidate redundant functions (`adjust_gamma_per_slice`, `adjust_brightness_per_slice`) into a single, rigorous pipeline (`correct_z_intensity_decay`). Enforce OLS-seeded NLLS for exponential models and fail loudly on fitting errors to prevent silent downstream propagation.

## 2025-05-25 - Hierarchical Dispersion Fallback in Robust Standardization
**Learning:** Robust standardization normally relies on Median Absolute Deviation (MAD), but for zero-inflated or highly tied data distributions, MAD can collapse identically to 0. A mathematically sound approach cascades through dispersion estimators (MAD -> Mean Absolute Deviation -> Standard Deviation). However, location and scale estimators must remain mathematically coupled during fallback (e.g., if Mean Absolute Deviation is used, it must center on the mean, not the median). This ensures the standardized distribution respects the correct asymptotic scaling properties, preventing mathematical incoherence or unprincipled epsilon divisions.
**Action:** Always use the centralized `eigenp_utils.stats.robust_standardize` function when zero-inflation or ties are expected. It correctly implements dimension-aware hierarchical dispersion fallback and mathematically couples location and scale estimators without arbitrary epsilon padding.

## 2025-05-25 - Multivariate Outlier Removal via Mahalanobis Distance
**Learning:** Using cascading univariate constraints (like z-scores or IQR per column) to remove multivariate outliers destroys underlying covariance structures by treating dimensions independently. This results in an orthogonal bounding box that fails to capture the true distributional shape. A mathematically correct approach uses the Mahalanobis distance with a pseudo-inverse covariance matrix, explicitly preserving structural relationships. Distances are strictly thresholded against Chi-Square distribution bounds, natively bypassing rows with `NaN`s without dropping valid covariance estimations.
**Action:** For multivariate outlier detection on structured datasets, implement and rely on `remove_outliers` using `method='mahalanobis'`, and define thresholds using cumulative probabilities (e.g. 0.99) mapped via the $\chi^2$ CDF, rather than heuristic univariate cutoffs.
